# SHiP work

## Collaboration entry points

Start from the public [SHiP website](https://ship.web.cern.ch/). It links to collaboration information, useful information, mailing lists, documents, meetings, logos, young-scientist resources, and the SHiP Twiki.

The public SHiP page describes the experiment as a search for feebly interacting long-lived particles and light dark matter, using two complementary detector systems and exploiting the high intensity of the SPS.

## Software entry points

Use the [SHiP software documentation](https://shipsoft.github.io/Documentation/) as the main public map. It currently covers:

- FairShip
- Simulation
- Geometry
- Geometry Service
- Field Service
- Data Model
- Packaging
- shipdist
- cvmfs_release
- HTCondor

The site is generated from Markdown in the public `ShipSoft/Documentation` repository, using Zensical and GitHub Pages.

## How to start a task

For a new student task, write down:

- The physics or software goal.
- The repository or documentation page involved.
- The expected output: note, plot, merge request, presentation, dataset, or validation.
- The environment: local, LXPLUS, container, CVMFS, or CI.
- The review path: supervisor, working group, or repository maintainers.

## Good first contributions

Good first contributions are usually small and observable:

- Fix outdated setup instructions.
- Reproduce a documented command sequence.
- Add a missing example.
- Run a simulation sample and record versions.
- Improve error messages or troubleshooting notes.
- Add a validation plot with a clear recipe.

Prefer small merge requests with a short explanation and reproducible commands.
